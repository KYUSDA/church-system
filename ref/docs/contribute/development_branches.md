# Development

To contribute:

1. Pull latest:

git pull origin `v2-staging`

1. Create a feature branch:

git checkout -b feature/`<team>`/`<description>`

1. Work on the task  
2. Push and submit PR into `v2-staging`  

3. Request a review from a teammate.

4. Merge after approval.

## 📌 Branch Strategy

### Main branches

- `main` → main branch
- `v2-staging` →development branch

### Create feature branches

feature/`<team>`/`<short-description>`

Examples:
feature/frontend/navbar
feature/backend/auth-update
feature/database/add-sermons-table

## ✔️ Pull Request Requirements

- Small PRs (50–200 lines ideal)
- Clear description  
- Screenshots for UI changes  
- Updated documentation if needed  
- Must pass linting & CI checks  

---

## 🚨 Forbidden Actions

- No direct commits to `main` or `v2-staging`
- No large PRs without discussion
- No secrets or credentials in commits
- No unreviewed schema changes
