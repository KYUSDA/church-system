## ♿ Accessibility First

This documentation is designed to be accessible for all team members and contributors.

### Accessibility goals

- Use **clear headings and structure** so pages are easy to scan.
- Follow **semantic HTML** and proper document outlines.
- Keep content readable with **plain language** and concise sections.
- Use **descriptive links** instead of generic text like “click here.”
- Support **keyboard navigation** and ensure page content works with assistive tools.

### Accessibility checklist for docs

- Ensure all images have **alt text** when added.
- Prefer **code blocks** and text layouts that are easy to read.
- Keep heading levels in order: `h1`, `h2`, `h3`, etc.
- Keep paragraphs short and use bullets for lists.
- Use high contrast text and avoid color-only meaning in diagrams or notes.

## 🛠 How to Use This Site

1. Start at this page to understand the overall structure.
2. Use the sidebar or quick links to jump to the version guide you need.
3. Read the project structure guides before making changes in `client/`, `server/`, or `backend_sanity/`.
4. Update or add new docs when services, deployment steps, or feature scope changes.
